# SDK Step 03: Streaming Responses

This example demonstrates how to stream responses in real-time as they're generated by the Nuclia API.

## Prerequisites

Before running this example, you need to set up your Nuclia credentials using .NET user secrets:

```bash
dotnet user-secrets set "ApiKey" "your-api-key-value"
dotnet user-secrets set "ZoneId" "your-zone-id-value"
dotnet user-secrets set "KnowledgeBoxId" "your-knowledge-box-id-value"
```

## Key Concepts

- Using `AskStreamAsync()` instead of `AskAsync()` for streaming responses
- Processing streaming chunks with `await foreach`
- Handling different content types in streaming responses
- Filtering and displaying answer text in real-time

## Why This Matters

Streaming provides a better user experience by:
- Showing results immediately rather than waiting for the entire response
- Providing visual feedback that processing is occurring
- Reducing perceived latency
- Enabling progressive rendering of content

## What This Example Does

1. Sets up the client using dependency injection
2. Sends a streaming request to the Ask API using `AskStreamAsync()`
3. Processes each chunk as it arrives using `await foreach`
4. Checks the type of each chunk (filtering for `AnswerContent`)
5. Displays answer text progressively (word by word or character by character)

## Code Highlights

```csharp
// Set up client with DI
var client = serviceProvider.GetRequiredService<INucliaDbClient>();

// Stream the response
AskRequest askRequest = new("Create a question about the data in this knowledge box?");
Console.WriteLine("Answer:");

await foreach (var chunk in client.Search.AskStreamAsync(askRequest))
{
    // Check chunk type and handle accordingly
    if (chunk.Item is AnswerContent answer)
    {
        Console.Write(answer.Text);
    }
}
```

## Understanding Streaming Chunks

Streaming responses can include multiple types of content:
- **AnswerContent**: The actual answer text (shown in this example)
- **RetrievalContent**: Retrieved resources and metadata (shown in Step 04)
- **CitationsContent**: Citation mappings (shown in Step 04)

Each answer chunk may contain a partial word or sub-word, so they're written sequentially to build the full answer text in real-time.

## Running This Example

1. Ensure your user secrets are configured (see Prerequisites above)

2. Navigate to this directory:
   ```bash
   cd samples/GettingStarted/Basics/SDK_Step03_Streaming_Responses
   ```

3. Run the application:
   ```bash
   dotnet run
   ```

You'll see the answer appear progressively as it's generated!
